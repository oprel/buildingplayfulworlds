Texture filter to apply when using custom shaders

Values:

Nearest

No smoothing, sample from a single nearest pixel only

Linear

Linear interpolation, averaged between neighbouring pixels

Public Functions

delegate bool FES.InputOverrideMethod(int button, int player, out bool handled)

Delegate for overriding input mapping

Return
Return true if the button is currently held down, or false if it is up
Parameters
  • button: The button being queried
  • player: The player for which the button is being queried
  • handled: Set to true if the button override was handled, or false if default FES mapping should be used

const int FES.FLIP_H = 1 << 0

Flip sprite horizontally

const int FES.FLIP_V = 1 << 1

Flip sprite vertically

const int FES.ROT_90_CW = 1 << 2

Rotate sprite 90 degrees clockwise

const int FES.ROT_90_CCW = ROT_90_CW | FLIP_H | FLIP_V

Rotate sprite 90 degrees counter-clockwise. This is equivalent to FES.ROT_90_CW | FES.FLIP_H | FES.FLIP_V

const int FES.ROT_180_CCW = FLIP_H | FLIP_V

Rotate sprite 180 degrees counter-clockwise. This is equivalent to FES.FLIP_H | FES.FLIP_V

const int FES.ROT_180_CW = FLIP_H | FLIP_V

Rotate sprite 90 degrees clockwise. This is equivalent to FES.FLIP_H | FES.FLIP_V

const int FES.ROT_270_CW = ROT_90_CW | FLIP_H | FLIP_V

Rotate sprite 270 degrees clockwise. This is equivalent to FES.ROT_90_CW | FES.FLIP_H | FES.FLIP_V

const int FES.ROT_270_CCW = ROT_90_CW

Rotate sprite 270 degrees counter-clockwise. This is equivalent to FES.ROT_90_CW

Property

property FES::Game

Reference to the Game running under FES. This is the same instance that is given to FES by the FES.Initialize method.

property FES::DisplaySize

Display size as given in IFESGame.QueryHardware.

property FES::MapSize

Tilemap size as given in IFESGame.QueryHardware.

property FES::MapLayers

Amount of map layers as given in IFESGame.QueryHardware.

property FES::FPS

Get target FPS as set by IFESGame.QueryHardware.

property FES::UpdateInterval

Get update interval which is equivalent to 1 / FES.FPS.

property FES::Ticks

Represents the count of calls to IFESGame.Update since startup, or since FES.TicksReset was called.

static void FES.SpriteSheetSetup(int index, string filename, Size2i spriteSize)

Load a sprite sheet and define the size of the sprites in it.

The size of sprites in the sprite sheet also determines with size of each tile in a tilemap layer that uses this sprite sheet.

Parameters
  • index: Sprite sheet slot index to load into
  • filename: Filename of the sprite sheet file, must be within a Resources folder
  • spriteSize: The size of sprites in this sprite sheet

class HardwareSettings

Defines hardware settings for the FES game. A FES Game fills out this structure in response to the call to IFESGame.QueryHardware.

Public Members

Size2i FES.HardwareSettings.DisplaySize = new Size2i(480, 270)

Display size, dimensions must be divisible by 2

Size2i FES.HardwareSettings.MapSize = new Size2i(256, 256)

Maximum tilemap size, in tiles. This number should be kept close to the maximum size required by your game, in order to conserve memory.

int FES.HardwareSettings.MapLayers = 8

Maximum amount of tilemap layers. This number should be kept close to the maximum layer count required by your game, in order to conserve memory.

ColorMode FES.HardwareSettings.ColorMode = ColorMode.RGB

Color mode. This can be either ColorMode.Indexed for an indexed mode with a strictly limited palette of colors, or ColorMode.RGB for any RGB color.

string FES.HardwareSettings.Palette = null

File name of the color palette. Ignored if HardwareSettings.ColorMode is ColorMode.RGB. Each pixel in the color palette is considered a color palette entry. The pixels are read from top-left to bottom-right. Maximum of 250 colors are supported in a palette, and so the color palette may not contain more than 250 pixels. The size of the color palette cannot be changed after the call to IFESGame.QueryHardware. The palette file must be located somewhere under a Resources folder.

int FES.HardwareSettings.FPS = 60

Target frames per second. Defaults to 60.

PixelStyle FES.HardwareSettings.PixelStyle = PixelStyle.Square

Display pixel style

interface IFESGame

Defines IFESGame interface. Every FES game must implement this interface.

Public Functions

HardwareSettings FES.IFESGame.QueryHardware()

Called once on startup to query the game for its hardware capabilities. This call happens before any other call to this interface.

Return
Hardware capabilities

bool FES.IFESGame.Initialize()

Called once after QueryHardware. The game should initialize it’s state here.

Return
Return true if initialization was successful, false otherwise

void FES.IFESGame.Update()

Called at a fixed interval of 60 times per second. Game logic goes in here.

void FES.IFESGame.Render()

Called to render to display, all draw APIs should be called here. Unlike IFESGame.Update this method is not called at a fixed framerate.
